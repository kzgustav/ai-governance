AIガバナンス方針書

第1章　目的と適用範囲
v1.0 · 2026年○月策定

1.1 目的

本方針書は、当社におけるAI（人工知能）技術の利用に関する基本方針を定め、AI活用によるビジネス価値の創出と、それに伴うリスクの適切な管理を両立することを目的とする。

本方針書は、NIST AI Risk Management Framework（AI RMF）の4機能（Govern・Map・Measure・Manage）を参考に構成し、経済産業省「AI事業者ガイドライン」およびJDLA「生成AI利用ガイドライン」の考え方を取り入れている。

1.2 適用範囲

本方針書は、当社の全役員および全従業員（契約社員、派遣社員、アルバイトを含む）に適用される。業務委託先がAIを用いて当社業務を遂行する場合も、本方針の趣旨を遵守するものとする。

1.3 対象とするAI

カテゴリ具体例
生成AI（テキスト）ChatGPT, Claude, Gemini, Copilot 等
生成AI（画像・動画）DALL-E, Midjourney, Stable Diffusion, Sora 等
コーディング支援AIGitHub Copilot, Cursor, Claude Code 等
業務特化型AI議事録AI, AI-OCR, AI翻訳, AIチャットボット 等
自社開発AIAPI連携で構築した社内ツール・BOT 等

基本方針（Govern）

当社は、AIの利用にあたり以下の基本方針を掲げる。

積極的活用の推進
AIは業務効率化・価値創出のための重要な手段であり、適切な統制のもとで積極的に活用する。
安全性の確保
個人情報・機密情報の保護を最優先とし、情報漏洩リスクを最小化する。
正確性の担保
AI生成物はあくまで補助的なものであり、最終的な判断と責任は人間が負う。
透明性の確保
AIの利用状況を可視化し、ステークホルダーへの説明責任を果たす。
継続的な改善
AI技術の進化と法規制の変化に合わせ、本方針およびルールブックを定期的に見直す。

推進体制

役割担当主な責務
最終承認者○○（役職）ガバナンス方針の最終承認、経営判断
推進リーダー○○（情報セキュリティ担当）ルールブック策定・更新、社内周知
推進メンバー各部門代表 ○名部門別ニーズ収集、運用フィードバック
外部アドバイザーai and humans inc.AI動向キャッチアップ、ルール更新支援、BOT構築・保守

報告・エスカレーションライン

通常時：現場担当者 → 部門推進メンバー → 推進リーダー → 最終承認者

緊急時：推進リーダーに直接報告し、既存の情報セキュリティインシデント対応フローに準ずる。

リスクの把握と評価（Map・Measure）

リスク分類概要既存セキュリティとの対応
情報漏洩機密データがAIサービス提供者のサーバーに送信されるISMS A.8（資産管理）の延長
ハルシネーションAIが事実と異なる情報を生成し、そのまま利用されるリスクAI固有
著作権侵害生成物が第三者の著作物に類似している可能性AI固有
プロンプトインジェクション悪意ある入力によりAIが意図しない動作をするSQLインジェクション対策と同様
シャドーAI従業員が未許可のAIツールを業務に使用シャドーIT管理ポリシーの拡張
バイアス・公平性AIの出力に偏りがあり差別的判断につながるAI固有

リスク管理と運用（Manage）

ルールブックとの連携

本方針書に基づく具体的な利用ルール、OK/NG基準、申請フローは、別冊「AI利用ルールブック」に定める。本方針書は経営方針レイヤー、ルールブックは現場運用レイヤーとして、二層構造で運用する。

ガバナンスBOTの運用

ルールブックの内容をベースとした「ガバナンスBOT」を社内に展開し、従業員がAI利用に関する判断に迷った際の一次相談窓口とする。

インシデント対応

（1）推進リーダーへの即時報告　（2）該当AIツールの利用一時停止　（3）影響範囲の調査　（4）再発防止策の策定とルールブックへの反映

教育・研修

全社員を対象としたAIリテラシー研修を年1回以上実施する。研修内容：

  • 本ガバナンス方針の理解
  • ルールブックの内容と運用方法
  • ガバナンスBOTの使い方
  • AI利用時のリスクと対処法
  • 最新のAI動向と法規制の変化

推進チームメンバーに対しては、より高度な内容の研修を別途実施する。

見直しと改訂

定期見直し：半期に1回（推進チームが起案し、最終承認者が承認）

臨時見直し：重大なインシデント発生時、法規制の重要な変更時、事業環境の大きな変化時

第2条　許可ツールリスト（ホワイトリスト）

以下のツールは、所定の利用条件を遵守する限り、業務利用を許可する。リストにないツールの利用は第6条の申請フローに従うこと。

ツール名契約形態用途注意事項
ChatGPTEnterprise / API文章作成・要約・翻訳・調査個人無料版の業務利用は禁止
ClaudePro / API文章作成・分析・コード生成APIは学習非使用を確認済
GitHub CopilotBusinessコーディング支援機密ロジックの入力に注意

第3条　データ入力ルール（OK / NG / 要申請）

データ分類判定条件・備考
公開情報（HP掲載内容等）OKそのまま入力可
社内一般情報（議事メモ等）OK社名を伏せることを推奨
取引先から受領した情報要申請NDA内容を確認の上、推進リーダーが判断
個人情報（氏名・連絡先等）NG匿名化・仮名化処理後であればOK
顧客の機密情報NG一切入力禁止
自社の機密情報（戦略・財務等）NG一切入力禁止
ソースコード（自社開発）要申請コアロジック以外は推進リーダー承認で可
他社の著作物（記事・書籍等）要申請要約目的の短文引用は可。全文入力はNG
判断に迷う場合は、ガバナンスBOTに質問するか、推進リーダーに確認すること。

第4条　生成物の取り扱いルール

4.1 ファクトチェック義務

AI生成物をそのまま社外に公開・送信してはならない。必ず以下のチェックを行うこと。

#チェック項目確認方法
1事実関係の正確性一次情報源で裏取りを行う
2数値・固有名詞の正確性原典と照合する
3著作権侵害の有無既存著作物との類似をチェック
4自社ブランドとの整合性トーン・表現が適切か確認
5個人情報の混入有無生成物に個人を特定できる情報がないか確認

4.2 社外公開時の追加ルール

AI生成物を社外に公開する場合は、上記チェックに加え、部門長の承認を必須とする。

4.3 著作権の考え方

AI生成物については著作権が発生しない可能性がある。重要な成果物については、人間の創作的関与を十分に加えた上で利用すること。

第5条　禁止される利用

以下の目的でのAI利用は、理由を問わず禁止する。

01人事評価・採用選考における最終判断をAIに委ねること
02法的拘束力のある文書（契約書等）をAI生成物のみで作成すること
03個人や特定の集団に対する差別的・攻撃的なコンテンツの生成
04他社の営業秘密や個人情報を推測・復元する目的での利用
05マルウェアやフィッシングコンテンツの生成
06ディープフェイク（虚偽の画像・動画・音声）の生成

第6条　申請フロー

6.1 新規AIツール導入申請

Step担当内容
1申請者AI利用申請書を記入（ツール名、利用目的、入力予定データの種類、想定リスク）
2推進メンバー一次レビュー（データ分類の妥当性チェック）
3推進リーダーセキュリティ観点での審査（利用規約、データ取扱いポリシーの確認）
4推進リーダー承認 / 条件付き承認 / 却下を判定し、申請者に通知
5推進リーダー承認時は許可ツールリストに追加し、全社に周知

6.2 判断に迷うケースの対応

まずガバナンスBOTに質問。BOTの回答で判断がつかない場合は推進メンバーまたは推進リーダーに相談する。

第7条　ガバナンスBOTについて

ガバナンスBOTは、本ルールブックの内容を学習した社内AIチャットボットです。以下の機能を提供します。

データ入力可否の判定
「このデータをChatGPTに入れていいですか？」→ OK / NG / 要申請を回答
ツール利用可否の判定
「○○というツールを使いたい」→ 許可リスト照合
ルールの参照
「社外公開時のルールは？」→ 該当条文を表示
エスカレーション
BOTが判断できない場合、推進チームへの問い合わせを自動起票
BOTの判断はあくまで補助的なものであり、最終的な責任は利用者および承認者にあります。

FAQ / よくある判断事例

従業員から寄せられることが想定される質問と回答です。ガバナンスBOTの初期学習データとしても利用します。

会議の議事録をChatGPTで要約してもいい？
社内一般情報であればOK。ただし、取引先名や個人名は伏せてから入力してください。NDA対象の情報が含まれる場合は推進リーダーに確認。
お客様から受け取った資料をAIに読み込ませて分析したい
原則NG。取引先の機密情報に該当する可能性があります。匿名化した上で利用する場合は要申請。
AIで作った提案書をそのままお客様に送っていい？
そのままはNG。必ずファクトチェック（第4条）を行い、部門長の承認を得てから送付してください。
個人のChatGPTアカウント（無料版）で業務の質問をしていい？
NG。無料版はデータが学習に使われる可能性があります。必ず会社契約のEnterprise版またはAPIを使用してください。
AIが生成した画像を社内プレゼンに使いたい
社内利用はOK。ただし、実在の人物に酷似した画像や他社のロゴ・キャラクターに類似した画像は使用禁止。社外発表に使う場合は要申請。
新しいAIツールを見つけた。試しに使ってみていい？
業務データを入れなければ個人の学習として試用可。業務で使いたい場合は第6条の申請フローに従ってください。
